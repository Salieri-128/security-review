# Repository Instructions

## Notes

When the user asks to create, supplement, organize, or update study notes, add the content under `notes/` in the most appropriate subdirectory.

Use the existing note style:

- short Chinese explanations
- clear sections such as `一句话理解`, `为什么重要`, `审计时注意`, and `我的理解`
- concise examples and audit-focused checklists

Choose note locations by topic:

- `notes/vulnerabilities/`: vulnerability categories, root causes, exploit paths, mitigations
- `notes/concepts/`: Solidity, EVM, transaction, encoding, and general security concepts
- `notes/defi/`: DeFi protocol mechanisms and designs, such as AMM models, Uniswap V3, vaults, lending, and LP accounting
- `notes/audit-methodology/`: audit workflow, review habits, severity, and finding writing
- `notes/attack-patterns/`: reusable exploit strategies and attacker playbooks
- `notes/foundry/`: Foundry, Forge, Cast, Anvil, cheatcodes, fuzzing, and invariant testing
- `notes/evm/`: lower-level EVM execution, storage, memory, calldata, and opcodes

Prefer adding or updating a focused Markdown file instead of scattering the same idea across multiple notes.
