# Notes

This folder is my personal smart contract security knowledge base.

The goal is not to copy course material. The goal is to build a searchable index of
concepts, vulnerabilities, tools, attack patterns, and review habits that I can reuse
while learning, auditing, writing findings, and preparing for interviews.

## Structure

- `concepts/`: core Solidity, EVM, and security concepts
- `vulnerabilities/`: vulnerability categories, root causes, exploit paths, and mitigations
- `audit-methodology/`: review workflow, checklists, severity, and finding-writing notes
- `evm/`: EVM behavior, opcodes, memory, storage, calldata, and execution details
- `foundry/`: Forge, Cast, Anvil, cheatcodes, fuzzing, and invariant testing
- `defi/`: DeFi primitives, protocols, accounting models, and risk patterns
- `resources/`: useful links, papers, tools, and reference material
- `attack-patterns/`: reusable exploit ideas and attacker playbooks

## Note Style

Prefer short, structured notes:

- What
- Why it matters
- Root cause
- Exploit path
- Impact
- Mitigation
- Example
- My understanding

Each note should make future review faster.
